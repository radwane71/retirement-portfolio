## 1. نظرة عامة على التطبيق

**الاسم:** ثروة  
**التعريف:** مفكرة حسابية شخصية لتدوين البيانات الاستثمارية  
**الجمهور المستهدف:** المستثمر السعودي الفردي الذي يريد إدارة ثروته بنفسه  
**اللغة:** عربية كاملة، اتجاه RTL  
**العملة:** الريال السعودي (ر.س) حصرياً  
**التنبيه القانوني (footer):** "أداة تقنية مستقلة للحساب الشخصي، لا تقدم خدمات مالية ولا توصيات استثمارية ولا ترتبط بأي جهة تنظيمية"

### الفلسفة الأساسية
- **لا بيانات خارجية:** كل الأرقام تُدخل يدوياً من المستخدم — لا API للأسعار
- **الخصوصية أولاً:** كل مستخدم يرى بياناته فقط (Row Level Security)
- **أداة حساب لا توصية:** الموقع يحسب ويعرض، القرار للمستخدم
- **شمولية:** من الأسهم إلى المدارس — "مكتب مالي شخصي"

### المجموعات الأربع في القائمة الجانبية
| المجموعة | الوصف | عدد الصفحات |
|---|---|---|
| 📈 محفظة التقاعد | الأسهم، المعاملات، الأرباح، التحليل | 11 صفحة |
| 💼 الإدارة المالية | صافي الثروة، مقسّم الراتب | 2 صفحة |
| 🏗️ استثمارات أخرى | العقارات، الصكوك | 2 صفحة |
| 🌱 إدارة الحياة | أهداف، مخزون، مدارس | 3 صفحات + رابط خارجي |

### 4.1 لوحة التحكم — `dashboard.html`

**الوصف:** المركز الرئيسي للتطبيق. يعرض صورة شاملة وآنية عن المحفظة الاستثمارية.

**بطاقات الإحصاء (5 صفوف × 4 بطاقات):**

**الصف 1 — قيمة الثروة:**
| البطاقة | الوصف |
|---|---|
| إجمالي قيمة المحفظة | الأسهم + النقد المتاح |
| نقد المحفظة | قابل للتعديل بنقرة واحدة |
| قيمة العقارات | مسحوب تلقائياً من صفحة العقارات |
| صافي الثروة | آخر لقطة محفوظة |

**الصف 2 — رأس المال والأداء:**
| البطاقة | الوصف |
|---|---|
| رأس المال المنشغل | تبويبان: صافي الشراء-البيع / طريقة الوسيط المرجح (WAC) |
| التكلفة مقابل القيمة السوقية | شريط تقدم + ر/خ محقق |
| الربح/الخسارة غير المحقق | بالريال والنسبة المئوية |
| ر/خ محقق من البيع | إجمالي صفقات البيع المغلقة |

**الصف 3 — الدخل:**
| البطاقة | الوصف |
|---|---|
| إجمالي الأرباح الموزعة | جميع السنوات |
| أرباح هذا العام (YTD) | السنة الحالية |
| عائد الأرباح | 4 تبويبات: متوقع (Forward) / مُسنوى / على التكلفة (YOC) / سوقي |
| صافي التدفق النقدي | هذا العام مع شريط تقدم |

**الصف 4 — التنويع والتكاليف:**
| البطاقة | الوصف |
|---|---|
| تفاصيل المحفظة | عدد الأسهم + عدد القطاعات + أسهم المنح |
| أعلى قطاع وزناً | اسم القطاع + النسبة |
| أقل قطاع وزناً | اسم القطاع + النسبة |
| التكاليف التراكمية | عمولات + ضريبة VAT |

**الصف 5 — الأداء والاستقلال المالي:**
| البطاقة | الوصف |
|---|---|
| العائد السنوي الحقيقي (XIRR) | معدل النمو السنوي المركّب |
| الدخل التوزيعي المتوقع | على أساس آخر 12 شهراً |
| تغطية الدخل السلبي للمصاريف | نسبة الأرباح/المصاريف المستهدفة |
| إجمالي الأصول الاستثمارية | أسهم + نقد + عقار + صكوك |

**المخططات البيانية:**
- **تخصيص الأصول (Asset Allocation):** دائري يوزع بين الأسهم/النقد/العقار/الصكوك
- **هدف الاستقلال المالي (FIRE):** شريط تقدم نحو هدف التقاعد مع تاريخ الوصول المتوقع
- **تحليل نقطة التعادل (Break-Even):** 4 أوضاع — ملخص / تفصيل / مساهمة / مخطط
- **توزيع القطاعات:** 3 أوضاع — دائري / أشرطة / بطاقات
- **وزن المحفظة — الحالي vs المستهدف:** 4 أوضاع — أشرطة / الفجوة / بطاقات / جدول (مع تلوين: أخضر=ضمن الهدف، أحمر=زيادة، أزرق=نقص، أصفر=بدون هدف)
- **مناطق الدخول والخروج:** بطاقة لكل سهم مع سعر الشراء والبيع المستهدف

**جدول الأسهم:**
- الأعمدة: الرمز، الاسم، القطاع، الأسهم، متوسط السعر، السعر الحالي، التكلفة، القيمة، ر/خ، الوزن، المستهدف
- **كل خلية قابلة للتعديل المباشر** بالنقر
- أزرار: تحديث الأسعار، تصدير CSV، مزامنة من المعاملات

**ميزة المزامنة من المعاملات:**
يعيد حساب عدد الأسهم ومتوسط التكلفة من سجل المعاملات — يحافظ على القطاع والهدف والسعر الحالي — يعرض مقارنة قبل/بعد قبل التأكيد.

### 4.2 سجل المعاملات — `transactions.html`

**الوصف:** تسجيل وإدارة كل عمليات شراء وبيع وأسهم المنح.

**الإدخال المجمّع (Bulk Entry):**
- جدول تفاعلي يمكن إضافة صفوف متعددة في آنٍ واحد
- حفظ كل الصفوف دفعةً واحدة بزر "إضافة معاملات (N)"
- مسح الكل بزر واحد
- يدعم التواريخ التاريخية بحرية

**إضافة معاملة واحدة:**
- نموذج مخفي افتراضياً (details/summary)
- حقول: التاريخ، رمز السهم، الاسم، النوع، عدد الأسهم، السعر
- الحقول المحسوبة تلقائياً: العمولة، الضريبة، الإجمالي
- تحذير عند إدخال رمز غير معروف في قاعدة البيانات

**أنواع المعاملات:**
- `buy` — شراء
- `sell` — بيع
- `grant` — أسهم منحة (لا عمولة عليها)

**حساب العمولة تلقائياً:**
```
العمولة = min(قيمة_العملية × 0.15%, 100 ر.س)
الضريبة = العمولة × 15%
الإجمالي (شراء) = أسهم × سعر + عمولة + ضريبة
الإجمالي (بيع)  = أسهم × سعر − عمولة − ضريبة
```

**شريط إحصائيات المعاملات:**
يظهر أعلى الجدول: إجمالي الشراء، إجمالي البيع، العمولات الكلية، VAT الكلي.

**الجدول الرئيسي:**
- ترتيب بالنقر على العناوين (تاريخ، رمز، اسم، نوع، أسهم، سعر، عمولة، إجمالي)
- تعديل مباشر لأي خلية
- عمود ✕ لحذف المعاملة مع تأكيد
- تصدير CSV

### 4.4 التدفقات النقدية — `cashflows.html`

- إجمالي السحوبات
- صافي التدفق (إيداعات − سحوبات)

**بطاقة الأرباح (مرتبطة تلقائياً):**
تعرض أرباح الموزعة من جدول الأرباح بدون حاجة لإعادة الإدخال.

**تسجيل الحركة:**
- التاريخ، النوع (إيداع/سحب)، المبلغ، ملاحظات

**جدول السجل:**
- فلتر بالسنة
- عرض كل الحركات مع إمكانية الحذف
- تصدير CSV

### 4.5 قاعدة بيانات أسهمي — `userdb.html`

**الوصف:** قاعدة بيانات مرجعية شخصية للأسهم التي يهتم بها المستخدم.

**الغرض:**
- إضافة الأسهم مرة واحدة برمزها واسمها وقطاعها
- تُستخدم لاحقاً في الإكمال التلقائي عند إدخال المعاملات والواتش ليست والأهداف
- مؤشر "في المحفظة" يُظهر أي الأسهم مملوكة فعلياً

**الحقول:**
- رمز السهم، الاسم، القطاع (من قائمة)

**الربط:**
يُغذي صفحة targets.html تلقائياً — أي سهم في قاعدة البيانات يظهر في صفحة الأهداف حتى لو لم يُشترى بعد (يظهر بمؤشر "📌 مخطط").

### 4.6 أهداف الأسهم والقطاعات — `targets.html`

**الوصف:** تحديد الأوزان المستهدفة لكل سهم وقطاع مع نظام تنبيهات للانحراف.

**جدول أهداف الأسهم:**
- الأعمدة: المهمة، الرمز، الاسم، القطاع، منطقة الشراء ≤، منطقة البيع ≥، الهدف %، الحالي %، الشريط، الحالة
- الوزن الحالي يُحسب تلقائياً من المحفظة
- شريط إجمالي يظهر مجموع الأهداف المحددة (يجب ألا يتجاوز 100%)

**جدول أهداف القطاعات:**
- الأعمدة: القطاع، الهدف %، الحالي %، الشريط، الحالة

**نظام الألوان (قابل للتخصيص من الإعدادات):**
- 🟢 أخضر: ضمن الهدف (الفارق ≤ 1% افتراضياً)
- 🟡 أصفر تنبيه: فارق بين 1-3%
- 🔴 أحمر: خارج الهدف (فارق > 3%)
- 🔵 أزرق: مخطط (في قاعدة البيانات لكن لم يُشترى بعد، وزن = 0%)

**مناطق الدخول والخروج:**
تُعدّ هنا وتُعرض في لوحة التحكم كبطاقة مستقلة.

### 4.8 الرؤية المستقبلية — `forecast.html`

**الوصف:** محرك توقعات متقدم يحاكي نمو المحفظة في 4 سيناريوهات بناءً على البيانات التاريخية الفعلية.

**البيانات التاريخية المستخرجة تلقائياً:**
- القيمة الحالية للمحفظة
- معدل نمو الأسعار التاريخي
- معدل عائد الأرباح الفعلي
- فترة الاستثمار الكلية

**السيناريوهات الأربعة:**
| السيناريو | اللون | الوصف |

### 4.12 صافي الثروة — `networth.html`

**الوصف:** الصورة الكاملة والحقيقية للثروة الصافية = الأصول − الالتزامات.

**معادلة صافي الثروة:**
```
صافي الثروة = إجمالي الأصول − إجمالي الالتزامات

الأصول = أسهم (تلقائي) + عقارات (تلقائي) + أصول أخرى (يدوي)
الالتزامات = قروض وبطاقات وأي مديونيات (يدوي)
```

**الأصول التلقائية:**
- قيمة الأسهم: تُسحب مباشرة من لوحة التحكم
- قيمة العقارات: تُسحب من صفحة العقارات (المملوكة والمؤجرة فقط)

**الأصول اليدوية (جدول قابل للتعديل):**
فئات مثل: حسابات بنكية، صكوك، مركبات، أثاث، ذهب... إلخ.

**الالتزامات اليدوية:**
فئات مثل: قرض عقاري، قرض سيارة، قرض شخصي، بطاقة ائتمان... إلخ.

**الرسوم البيانية:**
- رسم دائري لتكوين الثروة (أسهم / عقارات / أصول أخرى)
- 3 أوضاع: دائري / أشرطة / بطاقات

**مسار صافي الثروة عبر الزمن:**
- رسم بياني تاريخي من اللقطات المحفوظة
- 4 أوضاع: خط / أشرطة / مقارنة / جدول
- لقطات يُضيفها المستخدم يدوياً بتاريخ وقيمة وملاحظة

## 6. الحسابات والمنطق المالي

### حساب XIRR
```javascript
// طريقة Newton-Raphson مع fallback لـ Binary Search
// Input: مصفوفة من { date, amount } حيث الشراء سالب والقيمة الحالية موجبة
// Output: معدل العائد السنوي المعدّل بالزمن
```

### متوسط التكلفة المرجح (WAC)
```
WAC = Σ(عدد_الأسهم × سعر_الشراء) / Σ(عدد_الأسهم)
     لكل عمليات الشراء للسهم الواحد
```

### رأس المال الصافي المنشغل
```
Net Invested = Σ إجمالي الشراء − Σ إجمالي البيع
```

### العائد المُسنوى
```
Annualized Yield = (أرباح_آخر_12_شهر / قيمة_المحفظة_الحالية) × 100%
```

### العائد على التكلفة (YOC)
```
YOC = (أرباح_آخر_12_شهر / تكلفة_الشراء_WAC) × 100%
```

### العائد المتوقع (Forward Yield)
```
Forward Yield = (أرباح_آخر_12_شهر / القيمة_السوقية_الحالية) × 100%
```

### نقطة التعادل (Break-Even)
```
نقطة_التعادل = التكلفة_الكلية / عدد_الأسهم
نسبة_التعادل = (السعر_الحالي - نقطة_التعادل) / نقطة_التعادل × 100%
```

### حساب تغطية الدخل السلبي
```
نسبة_التغطية = الدخل_التوزيعي_السنوي / مصاريف_الحياة_السنوية × 100%
```

### محرك التوقعات (Compound Monthly)
```
القيمة_الشهرية_التالية = (القيمة_الحالية + DCA) × (1 + معدل_نمو_شهري)
                        + (القيمة × معدل_أرباح_شهري) × إعادة_الاستثمار
```
