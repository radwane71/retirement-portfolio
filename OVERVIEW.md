# ثروة — نظام تتبع المحافظ الشخصية الاستثمارية

> **حالة المشروع:** قيد التطوير — نسخة محلية (localhost)  
> **آخر تحديث:** مايو 2026

---

## 1. ما هو الموقع؟

**ثروة** أداة تتبع شخصية بالكامل. المستخدم يُدخل بياناته يدوياً، والموقع يعرضها ويحسب الإجماليات فقط.

**لا يوجد:**
- أي توصيات استثمارية
- أي ربط بأسعار أسهم أو بورصات أو APIs خارجية
- أي تقييم لأداء الأسهم (مرتفع / منخفض / شراء / بيع)
- أي مقارنة بمؤشرات السوق
- أي تنبيهات تحليلية بأي شكل

**السبب القانوني:** هذا التصميم يُبقي الموقع في خانة "أداة تتبع شخصية" وليس "منصة استثمارية"، وبالتالي لا يقع تحت رقابة هيئة السوق المالية السعودية ولا يحتاج أي ترخيص مالي.

---

## 2. التقنيات المستخدمة

| العنصر | التقنية |
|---|---|
| الواجهة الأمامية | HTML + CSS + JavaScript (Vanilla) — بدون أي framework |
| قاعدة البيانات | Supabase (PostgreSQL) |
| المصادقة | Supabase Auth (بريد إلكتروني + كلمة مرور) |
| الرسوم البيانية | Chart.js 4.4 |
| الخطوط | Google Fonts — Tajawal |
| التشغيل المحلي | Node.js HTTP server بسيط (`server.js`) |

> **لا توجد تراخيص مدفوعة.** كل المكتبات مجانية ومفتوحة المصدر. Supabase له طبقة مجانية كافية للاستخدام الشخصي.

## 4. قاعدة البيانات — الجداول

جميع الجداول مؤمّنة بـ **Row Level Security (RLS)** — كل مستخدم يرى بياناته فقط.

| الجدول | الوصف | الحقول الرئيسية |
|---|---|---|
| `holdings` | الأسهم في المحفظة | ticker, name, sector, shares, avg_price, current_price, target_weight |
| `transactions` | سجل الشراء والبيع | date, ticker, name, type (buy/sell), shares, price, commission, vat, total |
| `dividends` | الأرباح الموزعة المستلمة | date, ticker, name, amount, month, year |
| `cash_flows` | التدفقات النقدية السنوية | year, planned_amount, actual_amount |
| `net_worth_snapshots` | لقطات صافي الثروة | date, total_value, notes |
| `real_estate` | الأصول العقارية | name, type, purchase_value, current_value, status, monthly_rental, purchase_date |

## 5. الصفحات — تفصيل كامل

### 5.1 صفحة الدخول (`index.html`)
- **تسجيل الدخول:** بريد إلكتروني + كلمة مرور
- **إنشاء حساب:** بريد إلكتروني + رقم جوال (مع مفتاح الدول بالأعلام، اختياري) + كلمة مرور + تأكيد
- **نسيت كلمة المرور:** يُرسل رابط استرداد عبر Supabase للبريد الإلكتروني
- **إعادة تعيين كلمة المرور:** عند الضغط على رابط الإيميل يظهر فورم يعرض بريد المستخدم ويطلب الباسورد الجديد وتأكيده
- **بعد الدخول:** توجيه تلقائي لـ `dashboard.html`

### 5.2 لوحة التحكم (`dashboard.html`)
**بطاقات الإحصاء (الصف الأول):**
- إجمالي القيمة السوقية (الأسهم × السعر الحالي)
- إجمالي ما تم استثماره (من سجل المعاملات — نوع شراء)
- الربح/الخسارة غير المحقق (بالقيمة والنسبة)
- صافي الثروة (آخر لقطة مسجلة)

**بطاقات الإحصاء (الصف الثاني):**
- إجمالي الأرباح الموزعة (جميع السنوات)
- أرباح السنة الحالية
- قيمة العقارات الإجمالية
- الحقن النقدي هذا العام (مع شريط تقدم مخطط/فعلي)

**الرسوم البيانية:**
- توزيع القطاعات (Pie Chart)
- وزن المحفظة الحالي مقابل المستهدف (Bar Chart)

**جدول الأسهم:**
- يعرض: الرمز، الاسم، القطاع، الأسهم، متوسط السعر، السعر الحالي، التكلفة، القيمة، ر/خ، الوزن الحالي، المستهدف
- **تعديل مباشر:** كل خلية قابلة للنقر والتعديل دون فتح نافذة
- إضافة سهم جديد عبر نافذة Modal

### 5.3 سجل المعاملات (`transactions.html`)
- **إدخال مجمّع:** إضافة عدة معاملات في جدول واحد قبل الحفظ
- **إضافة واحدة:** فورم سريع لمعاملة واحدة
- **حساب تلقائي للعمولة:** 0.15% (بحد أقصى 100 ر.س) + ضريبة قيمة مضافة 15%
- **سجل كامل:** عرض جميع المعاملات مع إمكانية التعديل المباشر بالنقر
- تواريخ تاريخية مسموح بها بحرية

### 5.4 الأرباح الموزعة (`dividends.html`)
- تسجيل أرباح لكل سهم (التاريخ، الرمز، الاسم، المبلغ، الشهر، السنة)
- **ملخص سنوي:** إجمالي الأرباح لكل سنة
- **ملخص لكل سهم:** إجمالي ما استُلم من كل ورقة مالية
- سجل كامل مع تعديل مباشر بالنقر

### 5.5 التدفقات النقدية (`cashflows.html`)
- تسجيل المبلغ المخطط والفعلي لكل سنة
- **بطاقات الإحصاء:** إجمالي المخطط، إجمالي الفعلي، نسبة الإنجاز الكلية مع شريط تقدم
- جدول سنوي يعرض: السنة، المخطط، الفعلي، الفرق، نسبة الإنجاز
- تعديل مباشر بالنقر

### 5.6 صافي الثروة (`networth.html`)
- إضافة "لقطة" تمثل قيمة الثروة الإجمالية في تاريخ معين مع ملاحظات
- **رسم بياني خطي:** يُظهر مسار نمو الثروة عبر الزمن
- جدول اللقطات مع عرض التغيير بين كل لقطة والتالية
- تعديل مباشر بالنقر

### 5.7 العقارات (`realestate.html`)
- إضافة عقار جديد: الاسم، النوع، قيمة الشراء، القيمة الحالية، الحالة، الإيجار الشهري، تاريخ الشراء
- **أنواع العقارات:** شقة سكنية، فيلا، أرض، مكتب تجاري، محل تجاري، مستودع، أخرى
- **حالات العقار:** مملوك، مؤجر، مباع
- **بطاقات الإحصاء:** إجمالي القيمة الحالية، إجمالي تكلفة الشراء، الربح/الخسارة الإجمالي، الدخل الإيجاري الشهري
- جدول جميع العقارات مع تعديل مباشر

### 5.8 الإعدادات (`settings.html`)
- **تصدير نسخة احتياطية:** يُصدّر جميع البيانات في ملف JSON واحد باسم `portfolio_backup_YYYY-MM-DD.json`
- **استعادة من نسخة احتياطية:** استيراد ملف JSON — تحذير: يحذف البيانات الحالية ويستبدلها
- **معلومات الحساب:** عرض البريد الإلكتروني، معرّف المستخدم، تاريخ إنشاء الحساب
- زر تسجيل الخروج

## 6. الأمان

- **Row Level Security:** كل مستخدم يصل لبياناته فقط — مفعّل على مستوى قاعدة البيانات
- **لا كلمات مرور مخزنة في الكود:** المصادقة كلها عبر Supabase Auth
- **لا بيانات مشتركة بين المستخدمين:** كل جدول مرتبط بـ `user_id`
- **HTTPS في الإنتاج:** Supabase يوفرها تلقائياً

## 8. إعداد Supabase (للتشغيل لأول مرة)

1. إنشاء مشروع جديد على [supabase.com](https://supabase.com)
2. فتح **SQL Editor** ولصق محتوى `supabase_schema.sql` كاملاً وتنفيذه
3. نسخ **Project URL** و **anon public key** من Settings → API
4. وضعهما في `js/supabase.js`
5. في Authentication → Email Templates: تعديل قالب إعادة تعيين كلمة المرور (اختياري)
